Section 01

Executive Summary

TuitionMedia is a full-stack web platform that digitises the private tutoring ecosystem in Bangladesh — reducing dependency on intermediaries, accelerating tutor-student matching, and providing a trustworthy environment backed by profile verification and commission management. — Project Vision Statement

Private tutoring is a multi-billion BDT industry in Bangladesh, yet the matching process between students and tutors remains largely manual, slow, and trust-deficient. Parents spend days posting on social media groups and calling friends, while qualified tutors have no reliable way to showcase their credentials.

TuitionMedia addresses this gap by providing a centralised, verified marketplace where students post tuition requirements, tutors apply with complete profiles, and administrators oversee the platform's integrity. The system is further enhanced with an AI-powered assistant (Guru) for smart matching, a built-in messaging system, and integration with local payment gateways (bKash and Nagad) for transparent financial transactions.

Section 02

Problem Statement

The current private tutoring landscape in Bangladesh suffers from several interconnected problems:

🔍 Lack of Discovery

Students and tutors rely on word-of-mouth, Facebook groups, and physical notice boards. There is no single, structured platform where verified profiles are discoverable.

🛡️ Absence of Verification

Anyone can claim to be an expert tutor. Without ID verification and academic credential checks, families cannot make informed, safe hiring decisions.

💬 Communication Gaps

Negotiations happen on untracked, external channels (WhatsApp, phone calls). There is no record of conversations, no dispute mechanism, and no structured request flow.

💰 Opaque Financial Transactions

Commission and fee payments are cash-only and untracked. Agencies take large cuts with no digital receipt or accountability trail.

Section 03

Project Objectives

  • 🎯
    Build a Verified Marketplace Create a platform where student and tutor profiles are manually reviewed and approved by administrators before becoming publicly visible.
  • ⚡
    Accelerate Tutor-Student Matching Provide structured post and application flows so both parties can find and evaluate each other in minutes rather than days.
  • 🤖
    AI-Assisted Discovery via Guru Integrate the Gemini/Groq-powered Guru assistant to recommend tutors, answer questions, and provide platform insights to admins.
  • 💳
    Digitise Financial Transactions Integrate bKash and Nagad payment gateways to handle contact-unlock fees and tutor commission payments with full audit trails.
  • 🔔
    Real-time Notifications & Messaging Enable in-platform chat with unread badge tracking, push notifications via VAPID/Web Push, and SMS OTP for secure authentication.
  • 🛡️
    Comprehensive Admin Control Give administrators tools to approve/reject profiles and posts, ban bad actors, track commissions, and communicate directly with any user.
Section 04

Scope & Boundaries

✅ In Scope

  • User registration with phone + OTP
  • Role-based dashboards (Student / Tutor / Admin)
  • Tuition post creation, approval, browsing
  • Tutor profile creation, approval, browsing
  • Tuition request send/accept/reject workflow
  • In-platform messaging (admin + users)
  • bKash & Nagad payment integration
  • Commission tracking & proof upload
  • Push notification system (VAPID)
  • Guru AI assistant (Gemini / Groq)
  • Admin panel: approvals, bans, reporting
  • REST API for all core resources

❌ Out of Scope

  • Video/audio tutoring sessions
  • Automated tutor background checks (3rd party)
  • Mobile native apps (iOS / Android)
  • Multi-language interface (Bangla UI)
  • Automated salary disbursement to tutors
  • Review & rating system
  • Subscription-based premium tiers
  • Multi-city/international expansion logic
Section 05

Proposed Solution

TuitionMedia is architected as a monolithic Django web application with a well-structured set of decoupled apps, each owning its domain. A supplementary REST API (Django REST Framework) serves mobile/third-party clients.

Core Modules

postsTuition ListingsCreate/edit posts, admin approval, browse tutors & posts
tuitionsEngagement FlowTuition requests, active tuition management, salary tracking
paymentsTransactionsbKash/Nagad integration, contact unlock, commission payment
chatMessagingPermission-gated messages, admin inbox, unread counts, AJAX polling
guruAI AssistantGemini/Groq-powered chatbot, admin analytics mode
admin_panelAdministrationApprovals, user management, payment confirmation, reporting

Revenue Model

Contact Unlock Fee

Students pay a flat fee to unlock a tutor's phone number and address, creating a pay-per-lead monetisation model.

30% Commission

Tutors pay a 30% commission from their first month's salary upon starting an engagement, trackable via proof upload or direct payment.

Section 08

Project Timeline

Phase 1 — Foundation (Weeks 1–3)

Project setup, user authentication (phone + OTP), role-based access, basic profile management, database schema.

Phase 2 — Core Matching (Weeks 4–6)

Tuition post creation, admin approval workflow, browse tutors/posts, tuition request send/accept/reject, basic dashboard.

Phase 3 — Payments & Transactions (Weeks 7–9)

bKash/Nagad integration, contact unlock payment, commission payment and proof upload, payment history views.

Phase 4 — Communication (Weeks 10–11)

In-platform messaging with permission gates, admin inbox, AJAX polling, push notification system with VAPID.

Phase 5 — Intelligence & Polish (Weeks 12–13)

Guru AI integration (Gemini/Groq), admin analytics mode, notifications panel, REST API completion, responsive UI polish.

Phase 6 — Testing & Deployment (Week 14)

End-to-end testing, security hardening, static file collection, production deployment to Render/Gunicorn.
